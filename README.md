# user-report-db

Сервис для хранения и выдачи пользовательских взаимодействий внутри системы `ZOV`. Приложение принимает события вида `report`, `like` и `dislike`, сохраняет их в отдельное хранилище и предоставляет API для последующего получения истории взаимодействий.

## Что делает проект

- принимает жалобы пользователей;
- сохраняет лайки и дизлайки между пользователями;
- формирует историю отправленных и полученных взаимодействий;
- предоставляет HTTP API для интеграции с frontend и backend;
- использует колонночное хранилище для работы с данными взаимодействий.

## Технологии

- `Go`
- `net/http`
- `FrostDB`
- `Apache Arrow`
- `JSON API`

## Запуск

1. Установить зависимости:

```bash
go mod tidy
```

2. Запустить сервис:

```bash
go run main.go
```

Сервис запускается на порту `8069`.

## API

### `POST /app/report`

Создает запись о жалобе пользователя.

Пример тела запроса:

```json
{
  "userId": "user123",
  "reportedUserId": "user456",
  "message": "Spam in chat"
}
```

### `POST /app/like`

Создает запись о положительном взаимодействии.

Пример тела запроса:

```json
{
  "userId": "user123",
  "reportedUserId": "user456"
}
```

### `POST /app/dislike`

Создает запись о негативном взаимодействии.

Пример тела запроса:

```json
{
  "userId": "user123",
  "reportedUserId": "user456"
}
```

### `GET /app/user/{userId}/interactions/{direction}`

Возвращает историю взаимодействий пользователя.

- `direction=sent` — отправленные действия;
- `direction=received` — полученные действия.
